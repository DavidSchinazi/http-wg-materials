IETF HTTP WG 2022-02 Interim - QUERY Method

Julian Reschke, greenbytes

Current Status

Latest changes:

  • We have picked QUERY as method name (instead of the initial proposal to use SEARCH).
  • Cacheability is defined as similar to GET, except that the request content (payload) needs to be considered in addition to URI.

Open Questions

  • Can we assign a GETtable URI to the query result? Maybe Content-Location? (#1745).
  • Examples: would be good to have something with a well-defined query language (incl. media type), and matching response format. (#1906)
  • How do conditionals interact with QUERY? (#1917)
